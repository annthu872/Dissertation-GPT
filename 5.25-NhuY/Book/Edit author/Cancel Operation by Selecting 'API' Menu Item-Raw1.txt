Test Case: Test Case 1
testCaseName: Cancel Edit Operation by Selecting 'API' Menu Item
objective: Verify that the book edit operation is canceled and no data is updated in the database when the 'API' menu item is selected before submitting the form.
testSteps:
  - Step 1: Navigate to the book list/details page.
  - Step 2: Click the 'Update' button next to the book you wish to update.
  - Step 3: On the book edit form, start entering or modifying the book details such as Title, Author, Genre, Description, and Publication date.
  - Step 4: Before clicking the 'Submit' button, select the 'API' menu item.
expectedResult: The book edit operation is canceled, no data is updated in the database, and the user is redirected to the API page without saving any changes made to the book details.
explanation: This test case ensures that when a user decides to cancel the edit operation by navigating away from the edit page through the 'API' menu item, the operation is effectively canceled, and no changes are committed to the database. It verifies the application's ability to handle navigation actions as a form of operation cancellation during the edit process.
