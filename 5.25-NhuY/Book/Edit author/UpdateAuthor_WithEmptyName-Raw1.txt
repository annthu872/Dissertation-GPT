Test Case: Test Case 1
testCaseName: Update Book with Empty Author Name
objective: Verify that the system prevents updating a book record with an empty author name and displays the correct error message.
testSteps:
  - Step 1: Navigate to the book list/details page.
  - Step 2: Click the 'Update' button next to the book you wish to update.
  - Step 3: In the edit form, clear the 'Author' field to make it empty.
  - Step 4: Fill in the rest of the fields with valid data.
  - Step 5: Press the 'Submit' button.
expectedResult: The system should display an error message indicating that the author name cannot be empty and the book record should not be updated in the database.
explanation: This test case ensures that the system enforces the requirement for an author name when updating a book record, preventing incomplete or incorrect data from being saved.

Test Case: Test Case 2
testCaseName: Cancel Update Operation on Book
objective: Verify that the update operation can be canceled without saving any changes to the database.
testSteps:
  - Step 1: Navigate to the book list/details page.
  - Step 2: Click the 'Update' button next to the book you wish to update.
  - Step 3: Start entering or modifying the data in the form fields.
  - Step 4: Press the 'Cancel' button before submitting the form.
expectedResult: The system should not save any changes to the database and should display the list of book records, showing that no updates have been made.
explanation: This test case verifies that users have the option to cancel the update operation, ensuring that accidental or unwanted changes are not saved.
